C'est quoi le clustering ?

Le clustering, c'est demander à l'ordinateur de regrouper les images qui se ressemblent sans lui dire quelles catégories existent. Il trouve les groupes tout seul en analysant le contenu visuel.

Étape 1 : Les embeddings (fiches d'identité)

L'ordi ne « voit » pas les images comme toi. Il faut d'abord transformer chaque image en une liste de nombres (un vecteur) qui résume son contenu. C'est ce qu'on appelle un embedding.

MéthodeComment ça marcheQuand l'utiliser
CNN (ResNet-18)Un réseau de neurones qui a déjà vu des millions d'images. Il sait extraire des caractéristiques visuelles (formes, textures, couleurs).Premier choix recommandé — rapide et efficace même sans entraînement
Auto-encodeurUn réseau entraîné spécifiquement sur TES images pour apprendre à les compresser/décompresser.Si tu veux des features adaptées à ton dataset (nécessite d'abord lancer scripts/train_autoencoder.py)
Conseil : commence toujours par CNN (ResNet-18). C'est le plus simple et ça marche bien dès le départ.

Étape 2 : L'algorithme de clustering

AlgorithmeComment ça marcheParamètres
K-Means Tu lui dis « je veux K groupes » et il cherche le meilleur découpage possible. K (nombre de clusters) : commence par 3 ou 4. Regarde le graphique « coude » pour trouver le K optimal — c'est le point où ajouter un cluster n'améliore plus beaucoup.
DBSCAN Il trouve les groupes tout seul en cherchant les zones denses. Les images isolées sont marquées comme « bruit ». Epsilon (rayon de voisinage) : 0.5 par défaut. Si trop de bruit → augmenter. Min samples : 2 par défaut (minimum de voisins pour former un groupe).
Semi-supervisé Utilise les labels que tu as donnés dans l'onglet Étiquetage pour guider le clustering. Fonctionne mieux si tu as déjà étiqueté quelques images.

Comment lire les résultats ?

  • Scatter plot UMAP — Chaque point = une image. Les points proches = images similaires. Les couleurs = les clusters. Si les groupes sont bien séparés, c'est bon signe.
  • Score silhouette — Entre -1 et +1. Plus c'est proche de 1, mieux les clusters sont séparés. Au-dessus de 0.5 = bon. En dessous de 0.25 = les groupes sont flous.
  • Davies-Bouldin — Plus c'est bas, mieux c'est. Compare la dispersion interne vs la distance entre clusters.
  • Graphique du coude — Montre l'inertie (erreur) pour chaque K. Le « coude » (changement de pente) indique le K optimal.
Attention : avec seulement 13 images, les métriques ne seront pas parfaites. C'est normal. L'inspection visuelle (regarder les images dans chaque cluster) est plus fiable que les chiffres.
